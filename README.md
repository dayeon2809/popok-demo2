# Piece of Cake 🍰
무용을 발견하는 가장 쉬운 방법

**Piece of Cake**는 공연을 고르고, 감상을 나누고, 아티스트와 단체를 발견하는 무용 커뮤니티 플랫폼입니다. 어렵게 느껴지는 무용 예술을 케이크 한 조각처럼 쉽고 가볍게 즐길 수 있는 환경을 만듭니다.

---

## 💻 서비스 소개

- **무용 공연 정보 탐색:** 다양한 무용 공연의 상세 정보와 일정을 쉽게 탐색할 수 있습니다.
- **공연 기록 및 감상 공유:** 관람한 공연에 대해 평점을 매기고 댓글을 달며 소통합니다.
- **아티스트와 무용단체 발견:** 흩어진 무용수, 안무가, 무용단의 기록을 아카이빙하여 세상에 알립니다.
- **대표작 중심의 아카이브:** 아티스트가 직접 대표작 정보와 포스터, 공연 영상을 등록하고 사용자들이 감상할 수 있습니다.

---

## ⚙️ 핵심 기능

- **공연 아카이브 및 탐색:** 상영 예정 공연과 지난 공연을 카테고리별로 정렬하여 조회하고, 포스터 및 상세 캐스팅/장소/티켓 링크 정보를 제공합니다.
- **아티스트 DB & 장르/유형 필터링:** 분야별(현대무용, 발레, 한국무용, 다원예술) 및 활동 유형별(안무가, 무용단, 프로젝트팀) 필터를 제공하여 편리한 아티스트 검색 기능을 지원합니다.
- **localStorage 기반 Mock Login 커뮤니티:** 무거운 서버 세션 및 로그인 게이트 없이, 브라우저 로컬 스토리지 기반의 간편 사용자 세션을 생성하여 즉각적인 공연 평점 등록, 한 줄 감상 댓글 작성, 관심 아티스트 및 공연 저장(북마크) 기능을 제공합니다.
- **아티스트 등록 신청 (`/submit`):** 신규 아티스트들이 본인의 이력 정보와 포트폴리오를 작성하여 가입 신청서를 제출할 수 있습니다.
- **대표작 포트폴리오 정보 등록:** 가입 신청서 내에서 여러 개의 대표작을 추가할 수 있으며, 각 작품마다 고유 정보(작품명, 역할, 이미지, 영상)를 작성합니다.
- **Supabase Storage 연동 이미지 업로드:** 포트폴리오 작성 중 사용자가 작품 이미지를 첨부하면 API를 통해 Supabase Storage의 전용 버킷에 파일을 업로드하고, 반환된 public URL을 데이터베이스의 `image_url` 필드에 동적으로 연결합니다.
- **제출 및 어드민 승인 프로세스:** 신청서가 제출되면 승인 대기 상태(`pending`)로 데이터베이스에 입력되며, 관리자가 패스코드로 보안 처리된 어드민 화면에서 내용을 검토 및 보완한 뒤 승인(`approve`) 처리를 진행하면 즉시 아티스트 DB로 연동되어 사이트에 공식 반영됩니다.

## 📄 주요 페이지

- **Home (`/`):** 서비스 가치 제안, 3D 로고 연출, 실시간 누적 현황 현황판(CountUp), 추천 아티스트 목록 및 최근 등록 공연 목록을 한눈에 조망하는 메인 페이지입니다.
- **Performances (`/performances`):** 전체 등록된 공연 정보들을 탐색하고 아카이브할 수 있는 목록 페이지입니다.
- **Performance Detail (`/performances/[id]`):** 특정 공연의 상세 설명, 일정, 장소, 캐스팅 정보를 확인하고 다른 사용자들과 댓글 및 별점을 통해 감상을 공유하는 페이지입니다.
- **Artists (`/artists`):** 전체 아티스트 정보들을 분야 및 유형에 맞게 필터 검색할 수 있는 데이터베이스 허브 페이지입니다.
- **Artist Detail (`/artists/[id]`):** 아티스트의 상세 프로필과 함께 AI Summary 요약 이력 정보, 대표작 포트폴리오 카드, 외부 웹진 및 소셜 채널의 리뷰/평론 목록을 볼 수 있는 핵심 화면입니다.
- **Submit (`/submit`):** 신규 안무가/단체가 자신의 기본 메타데이터 및 대표작 미디어를 첨부하여 플랫폼에 등록을 신청하는 가입 폼입니다.
- **Login (`/login`):** 닉네임과 이메일 주소만으로 손쉽게 가입하여 커뮤니티에 참여할 수 있게 해 주는 로컬 세션 활성화 페이지입니다.
- **Profile (`/profile`):** mock 로그인 세션을 기반으로 본인이 관심을 등록한 아티스트 및 공연 리스트, 과거 작성했던 감상 댓글과 평가 점수 목록을 조회하는 사용자 전용 마이페이지입니다.
- **Admin Dashboard (`/admin`, `/admin/submissions`, `/admin/artists`):** 패스코드로 안전하게 보호되는 관리자 패널로, 등록 신청 건들에 대한 조회, 수정, 반려 및 최종 승인 마이그레이션 처리를 주도합니다.

## 🎨 아티스트 포트폴리오 (Representative Works)

아티스트는 자신의 작업 세계를 나타내는 대표작을 입력하고 제출할 수 있습니다.

- **작품 기본 정보:** 각 대표작 객체는 작품명(`title`), 제작 연도(`year`), 참여 역할(`role`), 작품 소개 상세 설명(`description`)을 포함합니다.
- **작품 포스터 이미지 (`image_url`):** 신청 페이지에서 파일을 드래그/선택하여 Supabase Storage에 개별 업로드하면, 해당 작품의 고유 키값에 이미지 링크가 바인딩됩니다.
- **작품 영상 링크 (`video_url`):** 유튜브(YouTube) 및 비메오(Vimeo) 공연 영상 링크를 전달하면, 아티스트 상세 페이지 내 포트폴리오 카드를 클릭했을 때 해당 페이지 내에서 즉시 반응형 레이어로 구성된 임베드 비디오 팝업 플레이어가 동작합니다.
- **독립적인 이미지 관리:** 포트폴리오 카드 및 팝업 상세창의 포스터 이미지는 아티스트의 프로필 이미지와 완전히 격격되어 관리되며, 이미지를 올리지 않은 대표작은 아티스트 프로필 사진 대신 약속된 기본 플레이스홀더 케이크 이미지로 안전하게 대체 노출됩니다.

## 🔄 서비스 이용 흐름

### 1. 일반 사용자 흐름
1. **공연 발견:** 메인 페이지 혹은 `/performances`에서 보고 싶은 무용 공연을 탐색합니다.
2. **정보 확인 및 소통:** 공연의 일정, 장소를 파악한 뒤 관람 후 평점과 한 줄 코멘트를 작성해 생각을 공유합니다.
3. **아티스트 탐색:** 출연진/제작진 정보를 클릭하여 아티스트의 상세 프로필 페이지로 이동합니다.
4. **포트폴리오 감상:** 아티스트의 고유 대표작 카드를 확인하고, 유튜브/비메오 영상을 팝업으로 직접 재생해 보며 아티스트 고유의 움직임을 발견합니다.
5. **저장 및 보존:** 마음에 드는 아티스트나 공연은 프로필에 저장해 두고 언제든 편리하게 모아봅니다.

### 2. 아티스트 가입 및 노출 흐름
1. **신청서 작성:** `/submit` 페이지에서 인적 사항 및 포트폴리오(작품 정보, 포스터 업로드, 비디오 링크)를 입력합니다.
2. **스토리지 업로드:** 작품 이미지를 등록하면 백엔드 API를 통해 Supabase Storage의 `portfolio-images` 버킷에 보관됩니다.
3. **심사 대기:** 작성 완료 후 전송하면 Supabase `submissions` 테이블에 `pending` 상태로 신규 등록됩니다.
4. **관리자 승인:** 어드민이 관리자 패널에서 내용을 검토한 뒤 '승인' 처리를 내립니다.
5. **실시간 반영:** 승인 즉시 Supabase의 `artists` 테이블로 모든 데이터(포트폴리오 jsonb 데이터 및 이미지 경로 포함)가 안전하게 동기화 복사되어 서비스 전반에 즉각적으로 노출됩니다.

## 🛠 기술 스택

- **Core Framework:** Next.js 16.2.9 (Turbopack 개발 환경 지원)
- **UI library:** React 19.2.4 & React-DOM 19.2.4
- **Language:** TypeScript 5.0+
- **Styling:** Tailwind CSS 4.0 & Vanilla CSS
- **Database & Storage Backend:** Supabase Client SDK (`@supabase/supabase-js`, `@supabase/ssr`)
- **Data Synchronizer:** Airtable SDK (`airtable`)

---

## 🗄 데이터 및 이미지 관리 구조

- **Supabase DB (PostgreSQL):**
  - `submissions` 테이블: 아티스트 가입 신청에 대한 세부 메타데이터, 승인 상태(`status`), 대표작 정보(`portfolio_works` jsonb 구조)를 누적 기록합니다.
  - `artists` 테이블: 노출 승인이 완료된 정식 안무가 및 단체 레코드를 관리하며, 승인 시 모든 메타데이터와 포트폴리오 객체 리스트가 안전하게 이전 및 저장됩니다.
- **Supabase Storage:**
  - `portfolio-images` 버킷: 대표작 신청 시 업로드되는 고화질 무용 포스터 이미지들을 분리 보관하며, 브라우저가 안전하게 내려받을 수 있는 공용 URL을 서빙합니다.
  - `artist-media` 버킷: 아티스트 프로필 사진 및 기타 범용 미디어 파일들의 업로드를 격리하여 관리합니다.

## 🚀 실행 방법

### 1. 패키지 설치

```bash
npm install
```

### 2. 로컬 환경 변수 설정
프로젝트 루트 폴더에 `.env.local` 파일을 생성하고 아래 키를 입력해주세요. (실제 연결용 Supabase 및 Airtable 자격 증명이 요구됩니다.)

```env
# Supabase Configuration
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# Airtable Configuration (Legacy Cache Synchronizer)
AIRTABLE_API_KEY=pat...
AIRTABLE_BASE_ID=app...
```
> ⚠️ **중요:** 실제 환경 변수 키와 패스코드는 보안을 위해 절대 커밋하거나 외부에 공유하지 마십시오.

### 3. 로컬 개발 서버 실행

```bash
npm run dev
# → 브라우저에서 http://localhost:3000 으로 접속
```

### 4. 프로덕션 빌드 및 구동

```bash
npm run build
npm run start
```
